🍽️ TableTouch — Web-Based Restaurant Touchscreen Interface
TableTouch is a web-based touchscreen interface designed for restaurant tables, empowering guests to browse the menu, customize their orders, track preparation progress, and complete payment — all without needing to flag down a server. Built to modernize the dining experience and reduce wait times.

🎯 Purpose
TableTouch is designed to:

Give diners full control over their ordering experience
Eliminate unnecessary wait times for menus, ordering, and payment
Reduce the workload on serving staff during peak hours
Provide a seamless, intuitive interface optimized for touchscreen use


✨ Features

Menu Browsing — Explore the full menu with categories, item descriptions, and images
Order Customization — Modify dishes to suit dietary preferences, portion sizes, or add-ons
Order Progress Tracking — Real-time updates on preparation status, from kitchen to table
In-Table Payment — Complete the bill digitally without waiting for a server
No Server Interaction Required — Fully self-sufficient flow from browsing to checkout
Touchscreen-Optimized UI — Large tap targets, smooth navigation, and gesture-friendly design

▶️ Getting Started

Clone or download the repository to your local machine
Open the project folder
Launch index.html in any modern web browser (Chrome, Edge, Firefox, etc.)


For full functionality (order tracking, payment), ensure the backend server is running and properly configured.


🖥️ How It Works

Browse — The guest opens the interface at their table and explores the menu
Customize — Items are selected and personalized (extras, removals, special requests)
Order — The order is submitted directly to the kitchen
Track — A live progress indicator shows when the order is being prepared and is ready
Pay — The guest completes payment digitally and receives a receipt
